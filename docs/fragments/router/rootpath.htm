When and how to use rootPath

Whenever you use hash fragments is quite easy to control your fragments; if you decided to use pushState instead, the router checks location.pathname and try to resolve it

The place where your application is deployed affects location.pathname. If your app lives inside the server root like:
http://localhost/myapp/ => location.pathname = "/myapp/"

If instead the application lives deeper inside the server hierarchy, things change:
http://localhost/root/sub-root/myapp/ => location.pathname = "/root/sub-root/myapp/"

We can deal with this scenario by setting a rootPath: 

const myRouter = new luga.router.Router({
	pushState: true,
	rootPath: "/root/sub-root/myapp/"
});

Once a rootPath is set, the router will "substract" it from the current location.pathname and propery resolve the appropiated route
